Someone kept trying to break into my personal server. Automated probes, repeated logins, and dropped payloads. This was not a demo feed. That real problem started Server Guardian Warboard.

Defense came first. A read-only decoy watches the hostile automation without exposing the real host. Then, with Codex and GPT-5.6, I turned every filtered signal into an explainable scene.

The strategic map gives the live system a shape. Country-level aggregates and active routes replace endless rows of log data, so movement can be read at a glance. This is the public judging surface, a viewer, not an admin console, and it needs no account.

Privacy is enforced before anything reaches the browser. The backend turns source addresses into stable bot aliases, generalizes the host to Protected Sandbox, replaces login names with aliases, and defangs suspicious commands and URLs. The browser only ever sees structured, privacy-filtered events, never the raw attack record. NORAD shows that same safe stream as an operations-room view.

This product was built with Codex and GPT-5.6 during Build Week. The viewer modes, the privacy boundary, bilingual coverage, deterministic demos, and six-mode automated QA.

It also gives back. When an attack is confirmed high-confidence, the backend automatically files that address to the AbuseIPDB community feed. Defending one small server helps protect the whole internet.

One event stream now supports many ways to understand an attack. The RPG village turns activity into life, patrols, and encounters. A deterministic skirmish replays on demand, even when live traffic is quiet. The same signals stage a cinematic battle, and a first-person mode changes the viewpoint and weapon choreography. These are not separate mockups. They are views inside the same live, read-only application. At recording time it had visualized more than forty-eight thousand hostile automation events, across fourteen documented signal species, six deterministic judge modes, and three first-person weapon choreographies.

Every mode passes deterministic checks, and each deployed build is verified by matching its hash against the live demo.

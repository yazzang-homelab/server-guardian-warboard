Someone kept trying to break into my personal server. Automated probes, repeated logins, and dropped payloads. This was not a demo feed. That real problem started Server Guardian Warboard.

Defense came first. A read-only decoy watches hostile automation without exposing the real host. Then, with Codex and GPT-5.6, I turned every filtered signal into an explainable scene.

The strategic map gives the live system a shape. Country-level aggregates and active routes replace endless rows of log data, so movement can be read at a glance. This is a viewer, not an admin console, and it needs no account.

Privacy is enforced before anything reaches the browser. The backend turns source addresses into stable bot aliases, generalizes the host to Protected Sandbox, replaces login names with aliases, and defangs suspicious commands and URLs. The browser only sees structured, privacy-filtered events. NORAD shows that same safe stream without revealing the system behind it.

During Build Week, Codex and GPT-5.6 built the judge-facing product: the viewer modes, privacy boundary, bilingual interface, deterministic demos, automated QA, and a new community-reporting pipeline.

Here is that new feature. A confirmed payload or repeated brute-force attempt passes a high-confidence safety gate. The backend de-duplicates each address for twenty-four hours, enforces rate and daily limits, sends no victim data, and files only the attacker address and standard categories to AbuseIPDB.

The public app remains read-only, but now it proves the contribution. At this review render, the live counter shows two hundred reports, two hundred unique addresses, and ninety-eight percent average confidence, plus a badge whenever a new report is filed.

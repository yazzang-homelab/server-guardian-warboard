제출 순서

  1. 공개 저장소를 만들고 README, 소스, DEVPOST_SUBMISSION.md, docs/GPT-5.6-USAGE.md를 포함합니다.
  2. 저장소 전체에서 키·토큰·사설 IP·호스트명·원본 로그를 검색해 제거합니다.
  3. 아래 3분 스크립트로 화면을 녹화하고 영어 내레이션 또는 영어 자막을 넣습니다.
  4. Devpost 설명에 프로젝트 개요, 트랙, 데모 URL, repo URL, YouTube URL을 입력합니다.
  5. 제출 직전 시크릿 창에서 세 링크가 로그인 없이 열리는지 확인합니다.

Submission Order

  1. Create a public or judge-shareable repository containing the README, source, DEVPOST_SUBMISSION.md, and docs/GPT-5.6-USAGE.md.
  2. Scan it for keys, tokens, private addresses, hostnames, and raw logs.
  3. Record the three-minute flow below with English narration or captions.
  4. Add the overview, track, demo URL, repository URL, and YouTube URL to Devpost.
  5. Verify all three links in a signed-out browser before submitting.

공개 저장소에 넣지 말 것

Never Publish

API keys, .env, private IPs, real hostnames, SSH keys, service credentials, raw logs, account/request metadata, deployment inventory.

완료 조건

Definition of Done

데모, 저장소, 3분 영상이 모두 비로그인 상태에서 열리고 Devpost 제출이 완료되어야 최종 완료입니다. 현재 남은 외부 작업은 영상 업로드와 Devpost 제출입니다.

Completion requires an accessible demo, repository, three-minute video, and the final Devpost submission. Remaining external actions are video upload and the final Devpost submission.

This guide summarizes preparation work; the official rules remain authoritative.
